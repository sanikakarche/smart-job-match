Architecture

This system combines classical ML with an agentic LLM layer using Gemini 2.0 Flash native tool-calling.

POST /recommend
│
├─ Step 1: TF-IDF cosine similarity → top-5 jobs (Classical ML)
│        No external API · Zero latency · Discriminative scores
│
├─ Step 2: Gemini tool-call → parse_resume
│        Output: {name, skills, exp, roles, education}
│
├─ Step 3: Gemini tool-call → generate_match_explanations
│        Output: {explanations[], clarifying_question}
│
└─ Response: {candidate, ranked_jobs, clarifying_question, meta}

Why TF-IDF over sentence-transformers?
Vercel free tier has a 1024 MB memory limit. Large embedding models cause OOM on cold starts. TF-IDF runs in-memory, produces well-spread scores (not all 0.90+), and handles domain-specific vocabulary (BERT, LangChain, Kubeflow) well through IDF weighting.

Why two tool calls?
Separation of concerns — Step 1 focuses purely on understanding the candidate without job-data noise. Step 2 receives a clean structured profile, producing far more precise match reasoning than a single monolithic prompt.

Tech Stack: FastAPI · Gemini 2.0 Flash · TF-IDF · httpx · Pydantic · Vercel
